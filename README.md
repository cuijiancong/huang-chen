## 婚宴邀请函H5模板制作设计方案

### 一、项目策划阶段

#### 1.1 明确需求

- **页面数量**：建议4-6屏（首屏、爱情故事、婚礼信息、照片墙、感谢页）
- **核心功能**：倒计时、地图导航、背景音乐、相册轮播
- **交互方式**：滑屏翻页、点击弹窗、动画触发

#### 1.2 确定风格基调

| 风格类型 | 配色建议           | 适用场景           |
| :------- | :----------------- | :----------------- |
| 中式复古 | 正红 + 金色 + 暗红 | 传统婚礼、长辈较多 |
| 清新简约 | 白 + 绿 + 浅粉     | 户外婚礼、草坪婚礼 |
| 浪漫唯美 | 香槟金 + 浅粉 + 白 | 室内宴会、梦幻主题 |
| 暗调高级 | 深蓝 + 香槟金 + 黑 | 晚宴、轻奢风格     |

> 本方案选用：浪漫唯美 香槟金 + 浅粉 + 白 主题，婚纱照多为室内场景。

### 二、信息架构与页面设计

#### 页面1：封面（首屏）

- **核心元素**：新人名字（大标题）、婚期、一句Slogan（如“我们结婚啦”）
- **设计建议**：全屏背景图（婚纱照/插画），搭配金色或白色的优雅字体，底部加“上滑查看邀请函”提示动画（居中）

#### 页面2：爱情故事 / 邀请语

- **核心元素**：一段温暖的邀请文字 + 新人照片（可左右滑动）
- **设计建议**：文字居中或左对齐，背景用柔和的渐变或浅色纹理

#### 页面2：新郎页

- 核心元素：新郎大照片特写 + 新郎文字标识，文字整点忒特效
- 

#### 页面2：新娘页

- 核心元素：新娘大照片特写 + 新郎文字标识，文字整点忒特效
- 

#### 页面3：照片墙

- **核心元素**：4-9张精选婚纱照或生活照，网格或瀑布流排列
- **设计建议**：点击可放大查看，配合轻量级灯箱效果”

#### 页面4：婚礼信息

- **核心元素**：
  - 时间（含具体几点几分，如果时间还未到就显示“距离婚礼还有”如果时间已经过去了那就显示“我们已经结婚”）
  - 地点（酒店名称+详细地址）、
  - 流程（仪式/晚宴时间）
- **设计建议**：信息用卡片或列表清晰展示，附“点击导航”按钮（调用百度/高德地图）

#### 页面6：底部/感谢页

- **核心元素**：再次感谢语、背景音乐控制按钮（播放/暂停）、版权信息
- **设计建议**：简洁收尾，音乐按钮常驻右下角；放两个图片框主要用于展示新郎和新娘的收款二维码收取打赏的，

### 三、技术选型方案

#### 方案A：纯原生（推荐新手/轻量项目）

- **技术栈**：HTML5 + CSS3 + 原生JavaScript

### 四、开发流程

#### 4.1 环境准备

```
项目目录结构：
wedding-invite/
├── index.html          // 主页面
├── css/
│   └── style.css       // 样式文件
├── js/
│   └── main.js         // 交互逻辑
├── assets/
│   ├── images/         // 图片资源
│   ├── fonts/          // 字体文件
│   └── music/          // 背景音乐
└── README.md   // 本方案文档
```

#### 4.2 开发步骤

| 步骤 | 任务                                         | 预估时间 |
| :--- | :------------------------------------------- | :------- |
| 1    | 设计稿确认（手绘草图或Figma低保真）          | 1天      |
| 2    | 搭建HTML骨架，划分页面结构                   | 0.5天    |
| 3    | 编写CSS样式，实现设计稿视觉                  | 1-2天    |
| 4    | 实现JavaScript交互（翻页、动画、音乐、表单） | 1天      |
| 5    | 接入地图导航、表单数据收集（可选）           | 0.5天    |
| 6    | 真机测试 & 适配优化                          | 0.5天    |
| 7    | 部署上线                                     | 0.5天    |

### 五、测试与适配

| 测试项           | 检查要点                                     |
| :--------------- | :------------------------------------------- |
| **屏幕适配**     | 在iPhone、华为、小米等主流机型测试显示效果   |
| **横竖屏**       | 锁定竖屏（`orientation: portrait`）          |
| **网络加载**     | 弱网环境下图片懒加载、骨架屏占位             |
| **表单提交**     | 验证必填项、提交反馈、防重复提交             |
| **音乐播放**     | iOS需用户手势触发，Android部分机型需注意兼容 |
| **微信转发卡片** | 需要适配微信转发卡显示卡片连接，             |

### 七、素材准备清单

在开始开发前，建议先准备好以下素材，或可以先生成临时文件给于命名变量，后自行替换

- 新人照片（封面1张、故事页2-3张、照片墙6-9张）
- 背景音乐（MP3格式，建议2-3MB以内）
- 装饰性图标（爱心、花朵、戒指等，可用Font Awesome或Iconfont）
- 手写字体的Web版本（可选）
- 酒店Logo或场地实拍图（可选）
